# Alio — Design System Reference

> One-page reference for **colors, typography, radii, spacing, animations**.
> Source of truth lives in [`packages/theme/`](packages/theme/). This doc is generated by hand from those files — when you change a token, update both.
>
> **Product:** Alio · *"When you can't be there, Alio is."*
> **Stack:** Next.js + Tailwind CSS · Mobile-first (sm=640, md=768, lg=1024).

## Color system

All colors are exposed three ways:
- **TS constants** from `@alio/theme` → `colors.brand.primary` etc.
- **Tailwind classes** → `bg-brand-primary`, `text-gray-100`, `border-brand-border` etc.
- **CSS variables** in `globals.css` → `var(--color-brand-primary)`.

## Typography

### Font family

**Century Gothic** (Bold + Regular weights).

- Loaded locally via `next/font/local` from [`packages/theme/fonts/`](packages/theme/fonts/).
- Exposed as CSS variable `--font-century-gothic`.
- Fallback: `system-ui, sans-serif`.
- License note: the prototype uses a free-download version. Engineer should swap to a licensed copy for production.

### Type scale

Tailwind size names + their underlying values. Line-heights baked in.

| Tailwind | Size | Line height | Usage |
|---|---|---|---|
| `text-xs` | 10px | 14px | Tab labels, timestamps, "May 2026" calendar dates. |
| `text-sm` | 12px | 16px | Captions, secondary metadata, calendar day labels. |
| `text-base` | 14px | 20px | Body text, audio bubble transcripts, button labels. |
| `text-md` | 16px | 22px | Default body. Dropdown menu "voice"/"message", "View Notes" body. |
| `text-lg` | 18px | 24px | Section headings ("May 2026", "Upcoming"). |
| `text-xl` | 20px | 28px | Card titles ("Caregiver on the way", "Alio" pill, "Hi, I am listening"). |
| `text-2xl` | 24px | 30px | Hero text. "Harold is stable today.", placeholder page titles. |

### Weights

- `font-normal` (400) — body text, captions.
- `font-bold` (700) — every heading + button + label in this design system. (No 500/600 — we don't have Semibold; everything below "bold" maps visually to regular.)

### Text styles (semantic)

Use these names when designing or talking about a screen. Each row is a recipe: copy the Tailwind classes and paste into JSX.

| Style | Tailwind | Visual | Where it's used today |
|---|---|---|---|
| **Display** | `text-2xl font-bold` | 24/30, bold | Hero "Harold is stable today." on `TodayStatusCard`; placeholder page titles. The biggest text on a screen. One per screen max. |
| **Heading (H1)** | `text-xl font-bold` | 20/28, bold | Card titles — "Caregiver on the way" on `CaregiverStatusCard`, "Alio" brand pill, "Hi, I am listening". The primary headline of a card or section. |
| **Subheading (H2)** | `text-lg font-bold` | 18/24, bold | "May 2026" calendar header, appointment day number. Secondary heading inside a card. |
| **Section title (H3)** | `text-base font-bold` | 14/20, bold | "Upcoming" list header, caregiver name in expanded card, log list item titles. Smaller grouping headings. |
| **Button** | `text-base font-bold` | 14/20, bold, often white or `text-brand-active` | "Press to Speak", "Done", "View Notes". Always bold. |
| **Body** | `text-md` | 16/22, regular | Default paragraph copy. Currently sparse — used in dropdown "voice"/"message" label. Use for new long-form copy. |
| **Body small / Caption** | `text-sm text-gray-60` | 12/16, regular, muted | "Coming soon" subtitles, "Sarah Mitchell" subtitle under caregiver headline. Default for secondary metadata. |
| **Body small (bold)** | `text-sm font-bold` | 12/16, bold | Audio bubble transcript, task labels, calendar weekday letters + dates. Bold when emphasized or selectable. |
| **Eyebrow** | `text-sm font-bold text-white/70` (on dark) / `text-gray-60` (on light) | 12/16, bold, muted | "Today's Status" label above the hero on the purple card. Small label above a main heading. |
| **Caption (micro)** | `text-xs font-bold` | 10/14, bold | Vital chip label, date chip month ("May"), stepper step labels ("On Way", "Arrived"). |
| **Tab label** | `text-[11.5px] font-bold leading-none` | 11.5px, bold, tight | The TabBar labels only. Custom size sits between `xs` (10px) and `sm` (12px) to match Figma. |
| **Timestamp / Tabular** | `text-xs font-bold tabular-nums` | 10/14, bold, fixed-width digits | Audio bubble time "3:02", log history dates. `tabular-nums` keeps digits aligned. |

**Color pairing**: pair the style with the right color token from the [Color system](#color-system) section. Common combos:

- **Display / H1 / H2 / H3** → `text-gray-100` on light bg, `text-white` on brand or dark bg.
- **Body small** → `text-gray-60` (muted by default).
- **Eyebrow on brand-primary bg** → `text-white/70`.
- **Button on accent (lime)** → `text-brand-active`.
- **Button on alert (red)** → `text-white`.

**Line height**: most styles use Tailwind's defaults from the type scale. Override with `leading-tight`, `leading-snug`, or `leading-none` only when stacking text rows tightly (TabBar label, stepper label, vital chip).

2.5→ 10px
3  → 12px
4  → 16px
5  → 20px
6  → 24px
8  → 32px

Card padding is consistently `p-5` (20px). Section gaps are typically `gap-3` (12px).

## Animations

CSS keyframes in [`packages/theme/src/globals.css`](packages/theme/src/globals.css).

### `blob-pulse` (2.6s, infinite, ease-in-out)
- Applied to the gradient blob on the **Caregiver Logs voice screens** when recording.
- Animates `transform: scale(1 → 1.06 → 1.04 → 1)`, `opacity`, and `filter: blur + hue-rotate(0 → 20° → -15° → 0)` for a breathing + color-cycling effect.
- Class: `animate-[blob-pulse_2.6s_ease-in-out_infinite]`.

### `listening-gradient` (3.6s, infinite, ease-in-out)
- Applied to the **"Hi, I am listening"** text while recording.
- Animates `background-position` across a 300%-wide 5-stop gradient (`#1F2782 → #6F7FF5 → #F472B6 → #C0DA5A → #1F2782`).
- With `bg-clip-text text-transparent`, colors flow through the text glyphs.
- Class: `animate-[listening-gradient_3.6s_ease-in-out_infinite]`.

## Where things live in code

```
packages/theme/
├── src/
│   ├── tokens.ts         ← TS constants (colors, fontFamily, fontSize, borderRadius)
│   ├── tailwind-preset.ts← exposes tokens as Tailwind theme.extend
│   ├── globals.css       ← CSS variables + keyframes + base font-face binding
│   └── index.ts          ← public exports
└── fonts/
    ├── CenturyGothic-Regular.ttf
    ├── CenturyGothic-Bold.ttf
    └── COPYRIGHT.txt
```

Each Next.js app applies the preset via its `tailwind.config.ts`:

```ts
import { tailwindPreset } from '@alio/theme/tailwind-preset';
// ...
presets: [tailwindPreset as Config],
```

And imports the CSS once in the root layout:

```ts
import '@alio/theme/globals.css';
```

## When to add a new token

✅ **Promote to a token** when the value appears in **3+ places** OR when it is a brand-defining color.

❌ **Leave inline as `bg-[#hex]`** when it's a one-off illustration color (radial gradients, decorative effects) or a 1–2-use experiment.

When adding:
1. Add to `tokens.ts` under the right group (`brand`, `gray`, etc.).
2. Add to `tailwind-preset.ts` so `bg-token-name` works in Tailwind.
3. Add to `globals.css` as `--color-token-name` for CSS-level access.
4. Update this doc.
